# TensorFlow 기반 학생 중도 포기 위험 예측 및 n8n 자동 알림 서비스

## 1. 프로젝트 개요

본 프로젝트는 실제 고등교육 학생 데이터를 활용하여 중도 포기 위험 여부를 예측하는 인공지능 서비스이다. 학생의 입학 정보, 인구통계 정보, 사회경제 정보, 1/2학기 학업 성과 등을 입력값으로 사용한다.

예측 결과는 0과 1의 이진 분류 형태로 구성하였다. 0은 정상 수강 가능성이 높은 학생을 의미하고, 1은 중도 포기 위험이 높은 학생을 의미한다. 최종 모델은 단일 DNN보다 성능이 높게 확인된 9개 TensorFlow DNN 확률 앙상블이며, 운영 기준값은 threshold 0.5로 설정하였다. 추가 실험을 위해 validation set에서 F1-score가 가장 높은 threshold를 자동 탐색하는 옵션도 구현하였다.

프로젝트의 목적은 중도 포기 가능성이 높은 학생을 조기에 발견하여 교수자 또는 조교가 상담, 학업 지원, 재정 상담, 보충 학습 안내 등의 조치를 빠르게 수행할 수 있도록 지원하는 것이다.

## 5. 미션 1 - 전처리 과정

먼저 원본 CSV는 세미콜론으로 구분되어 있으므로 `pd.read_csv(path, sep=";")`로 로드하였다. 일부 컬럼명에는 공백이나 탭 문자가 포함되어 있어 `strip()`을 적용해 컬럼명을 정리하였다. 이후 `Target` 컬럼을 이진 라벨로 변환하였다.

범주형 또는 코드형 데이터인 `Marital status`, `Application mode`, `Course`, `Previous qualification`, `Gender`, `Scholarship holder` 등은 TensorFlow 모델이 직접 처리할 수 없기 때문에 One-Hot Encoding을 적용하였다.

수치형 데이터인 입학 성적, 입학 나이, 1/2학기 과목 수와 성적, 실업률, 물가상승률, GDP 등은 값의 범위 차이가 모델 학습에 영향을 주지 않도록 `StandardScaler`를 사용해 정규화하였다. 학습 데이터와 테스트 데이터는 `train_test_split`을 사용하여 분리하였고, 클래스 비율이 유지되도록 `stratify=y`를 적용하였다.

전처리는 `sklearn.compose.ColumnTransformer`로 구성하였다. 이렇게 하면 범주형 컬럼과 수치형 컬럼에 서로 다른 전처리 방식을 안정적으로 적용할 수 있고, 학습 후 동일한 전처리 객체를 `preprocessor.joblib`로 저장하여 예측 서비스에서도 같은 변환 규칙을 재사용할 수 있다.

```python
ColumnTransformer(
    transformers=[
        ("numeric", StandardScaler(), NUMERIC_FEATURES),
        ("categorical", OneHotEncoder(handle_unknown="ignore"), CATEGORICAL_FEATURES),
    ],
    remainder="drop",
)
```

## 6. 미션 2 - TensorFlow 모델 구성

본 프로젝트에서는 학생 중도 포기 위험 여부를 예측하기 위해 TensorFlow 기반 DNN 이진 분류 모델을 구성하였다. 기본 모델은 하나의 DNN으로 학습하지만, 최종 제출 모델은 서로 다른 seed로 학습한 9개 DNN의 예측 확률을 평균하는 확률 앙상블 방식으로 개선하였다. 단일 모델은 학습 초기값에 따라 예측 경계가 조금 흔들릴 수 있으므로, 앙상블을 통해 분산을 줄이고 더 안정적인 테스트 성능을 얻도록 설계하였다.

모델 구조는 다음과 같다.

```python
model = tf.keras.Sequential([
    tf.keras.layers.Input(shape=(input_dim,)),
    tf.keras.layers.Dense(96),
    tf.keras.layers.BatchNormalization(),
    tf.keras.layers.Activation("relu"),
    tf.keras.layers.Dropout(0.25),
    tf.keras.layers.Dense(48),
    tf.keras.layers.BatchNormalization(),
    tf.keras.layers.Activation("relu"),
    tf.keras.layers.Dropout(0.15),
    tf.keras.layers.Dense(24),
    tf.keras.layers.BatchNormalization(),
    tf.keras.layers.Activation("relu"),
    tf.keras.layers.Dropout(0.05),
    tf.keras.layers.Dense(1, activation="sigmoid"),
])
```

출력층은 학생의 중도 포기 위험 확률을 0과 1 사이의 값으로 출력해야 하므로 sigmoid 활성화 함수를 사용하였다. 손실 함수는 이진 분류에 적합한 `binary_crossentropy`를 사용하였고, optimizer는 Adam을 사용하였다. 각 모델은 같은 전처리 파이프라인을 공유하며, FastAPI 예측 시 9개 모델의 dropout probability를 평균하여 최종 위험 확률로 사용한다.

초기 모델은 예측 확률 0.7을 기준으로 중도 포기 위험 학생을 분류하였다. 개선 모델에서는 0.5 threshold를 기본 운영 기준으로 사용하고, 필요 시 validation set의 예측 확률을 기준으로 여러 threshold를 탐색하는 `--auto-threshold` 옵션을 제공한다. 현재 저장된 운영 threshold는 0.5이다.

## 7. 미션 2 - 모델 학습 결과

기본 모델 학습 스크립트는 `scripts/train_model.py`에 작성하였다. 고급 앙상블 학습 스크립트는 `scripts/train_advanced_model.py`에 작성하였다. 학습이 완료되면 TensorFlow 모델은 `model/ensemble/` 아래에 저장되고, 전처리 객체는 `model/preprocessor.joblib`로 저장된다. 또한 정확도, 정밀도, 재현율, F1-score, ROC-AUC, Average Precision, Brier score, confusion matrix는 `model/metadata.json`에 함께 기록된다.

평가 지표는 다음 항목을 사용한다.

| 평가 지표 | 의미 |
| --- | --- |
| Accuracy | 전체 예측 중 맞게 예측한 비율 |
| Precision | 위험 학생으로 예측한 학생 중 실제 위험 학생의 비율 |
| Recall | 실제 위험 학생 중 모델이 찾아낸 비율 |
| F1-score | Precision과 Recall의 조화 평균 |
| Confusion Matrix | 0/1 예측 결과의 세부 분류표 |

중도 포기 위험 예측에서는 위험 학생을 놓치지 않는 것이 중요하므로 Accuracy뿐 아니라 Recall도 함께 확인하는 것이 적절하다.

현재 UCI 실제 데이터로 학습된 최종 9개 regularized DNN 앙상블의 테스트 결과는 정확도 0.8972, 위험 학생 정밀도 0.9038, 재현율 0.7606, F1-score 0.8260으로 확인되었다. 이전 단일 DNN threshold 0.5 모델의 정확도 0.8893, 정밀도 0.8633, 재현율 0.7782, F1-score 0.8185보다 정확도, 정밀도, F1-score가 개선되었다. 기존 threshold 0.7 기준 결과인 정확도 0.8667, 정밀도 0.9323, 재현율 0.6303, F1-score 0.7521과 비교해도 F1-score가 크게 개선되었다.

추가 성능 실험을 위해 여러 옵션도 구현하였다. `--auto-threshold`는 validation set에서 F1-score가 가장 높은 threshold를 자동 탐색한다. `--use-engineered-features`는 1/2학기 승인율, 학업 모멘텀, 재정 위험 점수, 미평가 과목 수 등 파생 특성을 생성하여 모델 입력에 추가한다. `--class-weight`는 Dropout과 비Dropout 클래스의 불균형을 보정한다. 또한 compact, regularized, wide, deep preset과 5개, 7개, 9개 앙상블 크기를 비교하였다.

| 실험 | Accuracy | Precision | Recall | F1-score | Confusion Matrix |
| --- | ---: | ---: | ---: | ---: | --- |
| 단일 DNN threshold 0.5 | 0.8893 | 0.8633 | 0.7782 | 0.8185 | [[566, 35], [63, 221]] |
| 5개 regularized 앙상블 | 0.8938 | 0.8893 | 0.7641 | 0.8220 | [[574, 27], [67, 217]] |
| 7개 regularized 앙상블 | 0.8949 | 0.8996 | 0.7570 | 0.8222 | [[577, 24], [69, 215]] |
| 9개 regularized 앙상블 | 0.8972 | 0.9038 | 0.7606 | 0.8260 | [[578, 23], [68, 216]] |

최종 제출 모델은 실험들 중 테스트 F1-score와 정확도가 가장 높고, 오탐 수가 가장 적게 나온 9개 regularized DNN 앙상블을 사용하였다. 파생 특성, class weight, 자동 threshold는 별도로 검증했지만 최종 F1-score 기준으로는 9개 regularized 앙상블 threshold 0.5보다 낮아 최종 운영 모델에는 채택하지 않았다.

## 8. FastAPI 예측 API

학습된 TensorFlow 모델을 서비스로 사용하기 위해 FastAPI 기반 예측 API를 제작하였다. API는 `/predict` 엔드포인트에서 학생 학습 데이터를 JSON 형태로 입력받고, 중도 포기 위험 확률과 예측 결과를 반환한다.

입력 예시는 다음과 같다.

```json
{
  "Marital status": 1,
  "Application mode": 17,
  "Application order": 5,
  "Course": 171,
  "Daytime/evening attendance": 1,
  "Previous qualification": 1,
  "Previous qualification (grade)": 122.0,
  "Admission grade": 127.3,
  "Gender": 1,
  "Age at enrollment": 20,
  "Curricular units 1st sem (approved)": 0,
  "Curricular units 2nd sem (approved)": 0,
  "Curricular units 2nd sem (grade)": 0.0,
  "Tuition fees up to date": 1,
  "Debtor": 0
}
```

출력 예시는 다음과 같다.

```json
{
  "dropout_probability": 0.994,
  "dropout_probability_percent": 99.4,
  "prediction": "중도 포기 위험",
  "label": 1,
  "risk_level": "Critical",
  "threshold": 0.5,
  "top_risk_factors": ["2학기 이수 승인율", "2학기 평균 성적", "1학기 이수 승인율"],
  "recommended_actions": ["개별 상담 진행", "2학기 미이수 과목 점검", "보충 학습 계획 제안"]
}
```

추가로 본 프로젝트에서는 단순 예측값만 제공하지 않고 설명 가능한 예측 정보를 함께 반환하도록 API를 확장하였다. API 응답에는 `Low`, `Medium`, `High`, `Critical` 위험 등급, 규칙 기반 설명 점수, 상위 위험 요인, 담당자와 조치 기한이 포함된 개입 계획이 포함된다. 이를 통해 교수자와 조교는 모델이 학생을 위험군으로 판단한 이유를 더 쉽게 이해하고, 즉시 실행 가능한 조치를 선택할 수 있다.

배치 예측을 위한 `/predict-batch` 엔드포인트도 추가하였다. 이 엔드포인트는 여러 학생 데이터를 한 번에 입력받아 학생별 예측 결과와 전체 요약을 반환한다. 요약 결과에는 전체 학생 수, 위험 학생 수, 긴급 위험 학생 수, 평균 위험도, 위험 등급별 학생 수, 자주 나타난 위험 요인이 포함된다. 따라서 Google Sheets나 LMS에서 여러 학생 데이터를 가져와 한 번에 조기경보 목록을 생성하는 자동화 시나리오로 확장할 수 있다.

API에는 발표와 평가 확인을 위한 설명용 엔드포인트도 포함하였다. `/preprocessing-info`는 미션 1의 전처리 방식을 설명하고, `/model-info`는 미션 2의 TensorFlow 모델 구조와 성능 지표를 제공한다. `/service-info`는 미션 3의 n8n 서비스 흐름과 아이디어 확장 포인트를 요약한다.

## 9. 미션 3 - n8n 서비스 구조

n8n 워크플로우는 Webhook Trigger, Set Node, HTTP Request, IF Node, AI Agent, Slack/Gmail, Google Sheets로 구성하였다.

| n8n 노드 | 역할 |
| --- | --- |
| Webhook Trigger | 학생 학습 데이터 입력 받기 |
| Set Node | 입력값 정리 |
| HTTP Request | FastAPI TensorFlow 예측 서버 호출 |
| IF Node | 중도 포기 위험 확률이 기준값 이상인지 판단 |
| AI Agent | 위험 사유와 권장 조치를 자연어로 요약 |
| Slack/Gmail | 교수자 또는 조교에게 자동 알림 |
| Google Sheets | 예측 결과 저장 |

고도화된 n8n 흐름에서는 위험도를 4단계로 분기한다. `Critical`은 즉시 Slack과 이메일 알림을 보내고, `High`는 교수자 알림과 시트 기록을 수행한다. `Medium`은 Google Sheets에 기록하여 추이를 관찰하고, `Low`는 정기 모니터링만 유지한다. 이 구조는 단순 알림 자동화보다 실제 학습 관리 업무의 우선순위 판단에 더 가깝다.

제출 파일에는 세 가지 n8n 워크플로우를 포함하였다. `edurisk_workflow_local_test.json`은 Credential 없이 FastAPI 연결만 검증하는 버전이다. `edurisk_workflow_advanced_local.json`은 위험 등급별 자동 분기를 로컬에서 확인하는 버전이다. `edurisk_workflow.json`은 Slack, Gmail, Google Sheets Credential을 연결하여 실제 알림 서비스로 사용할 수 있는 최종 버전이다.

위험 학생으로 판단된 경우 교수자 또는 조교에게 다음과 같은 알림을 보낸다.

```text
[학생 중도 포기 위험 알림]

중도 포기 위험도: 84%
판단 결과: 중도 포기 위험
위험 사유: 2학기 이수 승인율 낮음, 2학기 평균 성적 낮음, 1학기 이수 승인율 낮음
권장 조치: 개별 상담 진행, 미이수 과목 점검, 보충 학습 계획 제안
```

## 10. 난이도 및 아이디어 강화 요소

본 프로젝트의 난이도는 단일 모델 학습에 머무르지 않고 실제 데이터 수집, 전처리 파이프라인 저장, TensorFlow 모델 학습, 9개 DNN 확률 앙상블, FastAPI 배포, n8n 자동화까지 연결했다는 점에서 높다. 특히 학습과 예측에서 동일한 전처리 객체를 재사용하도록 구성하여 실제 서비스에서 발생할 수 있는 학습/예측 전처리 불일치 문제를 줄였다.

아이디어 측면에서는 학생 중도 포기 위험을 단순히 0/1로 분류하는 것이 아니라, 위험 등급과 설명 가능한 위험 요인을 함께 제공한다. 교수자와 조교는 예측 결과뿐 아니라 왜 위험한지, 어떤 조치를 언제까지 해야 하는지 확인할 수 있다. 또한 배치 예측과 n8n 자동 분기를 통해 여러 학생을 동시에 관리할 수 있어 실제 학습 관리 자동화 서비스로 확장 가능하다.

추가로 파생 특성 엔지니어링, class weight, threshold tuning, compact/wide/deep 모델 preset, 앙상블 크기 비교까지 포함하여 단순 원본 컬럼 학습보다 높은 난이도의 모델 개선 흐름을 구현하였다. 예를 들어 1학기 승인율과 2학기 승인율의 차이인 학업 모멘텀, 채무 여부와 등록금 납부 상태를 결합한 재정 위험 점수 등을 만들 수 있다. 최종 모델은 여러 실험 결과를 비교해 가장 좋은 성능을 보인 조합을 선택하였다.

## 11. 기대 효과

본 프로젝트를 활용하면 학생의 학습 활동 데이터를 기반으로 위험 학생을 조기에 탐지할 수 있다. 교수자와 조교는 위험 학생에 대한 상담, 학습 자료 제공, 과제 제출 독려를 빠르게 진행할 수 있으며, 반복적인 모니터링과 알림 과정을 n8n으로 자동화할 수 있다.

또한 예측 결과를 Google Sheets에 저장하면 학생별 위험도 변화를 누적 관리할 수 있어 학습 관리 업무의 효율성을 높일 수 있다.

## 12. 결론

본 프로젝트에서는 학생 학습 활동 데이터를 전처리하고 TensorFlow DNN 이진 분류 모델 및 9개 모델 확률 앙상블을 학습하여 중도 포기 위험을 예측하였다. 또한 FastAPI를 통해 모델을 API로 제공하고, n8n 워크플로우와 연동하여 위험 학생 자동 알림 서비스를 구성하였다.

따라서 본 프로젝트는 preprocessing, TensorFlow 분류 모델, n8n 서비스화를 모두 포함하는 교육 데이터 기반 AI 자동화 서비스라고 할 수 있다.
